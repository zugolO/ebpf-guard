# Maintainers

This file lists the maintainers of the ebpf-guard project.

## What is a Maintainer?

Maintainers are individuals with commit access who are responsible for:
- Reviewing and merging pull requests
- Setting technical direction
- Making release decisions
- Enforcing the code of conduct
- Mentoring contributors

See [GOVERNANCE.md](GOVERNANCE.md) for details on how maintainers are selected.

## Becoming a Maintainer

We welcome new maintainers! To become a maintainer:

1. **Demonstrate sustained contribution** over at least 6 months
2. **Show good judgment** in code reviews and discussions
3. **Understand the codebase** across multiple components
4. **Be nominated** by an existing maintainer
5. **Be approved** by a majority of existing maintainers

If you're interested in becoming a maintainer, please:
- Start contributing (see [CONTRIBUTING.md](CONTRIBUTING.md))
- Participate in code reviews
- Join community discussions
- Reach out to existing maintainers

## Maintainer Responsibilities

### Code Review

- Review PRs within 5 business days
- Provide constructive feedback
- Ensure tests and documentation are included
- Verify DCO sign-off on all commits

### Releases

- Follow the release process in [GOVERNANCE.md](GOVERNANCE.md)
- Ensure all tests pass before releasing
- Write clear release notes
- Announce releases in community channels

### Community

- Respond to issues and discussions
- Enforce the [Code of Conduct](CODE_OF_CONDUCT.md)
- Welcome new contributors
- Mentor aspiring maintainers

### Security

- Respond to security reports within 48 hours
- Coordinate security fixes
- Issue security advisories when needed
