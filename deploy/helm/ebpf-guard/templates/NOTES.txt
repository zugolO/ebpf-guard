ebpf-guard has been deployed as a DaemonSet on every node.

** IMPORTANT: ebpf-guard requires a Linux kernel 5.15+ and runs privileged. **
   LSM enforcement features require kernel 5.7+.

Get the agent status:
  kubectl -n {{ include "ebpf-guard.namespace" . }} get daemonset {{ include "ebpf-guard.fullname" . }}

View logs from one node:
  kubectl -n {{ include "ebpf-guard.namespace" . }} logs daemonset/{{ include "ebpf-guard.fullname" . }}

Access the metrics endpoint (port-forward):
  kubectl -n {{ include "ebpf-guard.namespace" . }} port-forward daemonset/{{ include "ebpf-guard.fullname" . }} 9090:9090
  curl http://localhost:9090/metrics

View recent alerts:
  kubectl -n {{ include "ebpf-guard.namespace" . }} port-forward daemonset/{{ include "ebpf-guard.fullname" . }} 9090:9090
  curl -H "Authorization: Bearer <token>" http://localhost:9090/alerts

The bearer token is logged at agent startup if not set in values.yaml:
  kubectl -n {{ include "ebpf-guard.namespace" . }} logs daemonset/{{ include "ebpf-guard.fullname" . }} | grep "bearer token"
